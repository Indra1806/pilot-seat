## The quality of the output will depend heavily on **how you ask the topic**.

> You don't need to write long prompts every time. Use one of these patterns.

<div align=center>

# Pattern 1: New Topic (Recommended)

> When learning a topic from scratch:

```text
Topic: HTTP

Create documentation according to our documentation standard.
```

Example:

```text
Topic: Docker

Create documentation according to our documentation standard.
```

I will automatically generate:

* Introduction
* Why It Exists
* Problem It Solves
* Core Concepts
* Architecture
* Workflow
* Real World Usage
* Best Practices
* Common Mistakes
* Summary
* Glossary
* Keywords

</div>

# Special Commands

These shortcuts will save time.

### Beginner Mode

```text
Topic: Kubernetes

Mode: Beginner
```

Explain like someone seeing it for the first time.



### Professional Mode

```text
Topic: Kubernetes

Mode: Professional
```

Production-focused explanation.



### Book Mode

```text
Topic: Kubernetes

Mode: Book
```

Generates content suitable for future book/course material.



### GitHub Mode

```text
Topic: Kubernetes

Mode: GitHub
```

Produces README-ready documentation.

# Recommended Format for Chat

For most topics, simply ask:

```text
Topic: <Topic Name>

Mode: Mastery

Create documentation according to our documentation standard.
```

Example:

```text
Topic: Web Application Development

Mode: Mastery

Create documentation according to our documentation standard.
```

That single prompt will be enough for me to generate structured, GitHub-ready, future-book-ready documentation.
